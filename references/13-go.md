# Go Playbook

Use this file for Go services, CLIs, libraries, and tests.

## Read first
- `go.mod`, `go.work`, `golangci-lint`, `go test ./...`, CI, package layout.
- Existing package boundaries and interface ownership.

## Blockers
- Ignored errors, leaked goroutines, data races, nil pointer paths, or unbounded resource use.
- Context not passed through I/O, RPC, DB, queue, or long-running operations.
- SQL, command, path, template, or deserialization injection.
- Public API or wire-format changes without migration or compatibility handling.

## Review checks
- Return errors with enough context; do not log and return the same error at every layer.
- Define interfaces at the consumer side only when there is real variation.
- Keep concurrency ownership explicit: cancellation, channel close ownership, wait groups, locks.
- Prefer table tests for edge cases and regression tests for fixed bugs.
- Keep packages cohesive; avoid tiny pass-through packages that only rename calls.

## Repair moves
- Add context propagation and deterministic cancellation tests for I/O paths.
- Replace speculative interfaces with concrete types, or add real adapters when variation exists.
- Run `go test ./...` and race tests when concurrency changed.
